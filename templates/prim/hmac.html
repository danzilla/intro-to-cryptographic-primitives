Data Integrity and Authentication Algorithms

These are schemes based on the use of cryptographic algorithms designed to authenticate the identity of entities as well as to used to protect blocks of data, such as messages, from alteration

HMAC Hashed Message Authentication Code

The purpose of Hashed Message Authentication Code is to provide the means of authenticating the genuineness of data sent over a medium. In order to create the HMAC value you need three things: a hash algorithm like SHA-256, a secret key, and a message

Once the HMAC value is produce it is appended to the message which is then sent to the recipient. Once received the HMAC act as an authenticator to validate if the original data was modified or alter in transit. It does this by finding the HMAC value of the received message and compare it to the HMAC value received. If both HMAC values matches, then the message was not altered otherwise it was.
